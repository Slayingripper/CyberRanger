## Using This Documentation

### For New Users

1. Start with [Home](Home.md) to understand what CyberRanger is
2. Follow [Installation & Setup](Installation-and-Setup.md) to get it running
3. Read the [User Guide](User-Guide.md) to learn the basics
4. Explore [Topology Builder](Topology-Builder.md) or [Training System](Training-System.md) based on your needs

### For Developers

1. Review [Architecture](Architecture.md) to understand the system design
2. Check [API Reference](API-Reference.md) for endpoint documentation
3. Read [Contributing](Contributing.md) before making changes
4. Use [Troubleshooting](Troubleshooting.md) for debugging

### For Instructors

1. Learn the platform basics in [User Guide](User-Guide.md)
2. Study [Training System](Training-System.md) for creating courses
3. Use [Topology Builder](Topology-Builder.md) to design lab environments
4. Refer to [API Reference](API-Reference.md) for automation

## GitHub Wiki

These markdown files can be copied to a GitHub Wiki for the repository. To create a wiki:

1. Go to your GitHub repository
2. Click on the "Wiki" tab
3. Create pages with the same names as these files
4. Copy the content from each markdown file

## Updating Documentation

When updating the documentation:

1. Keep pages focused and concise
2. Use clear headings and structure
3. Include code examples where helpful
4. Add screenshots for UI features
5. Link between related pages
6. Test all commands and code snippets
